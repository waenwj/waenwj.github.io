组件提取策略

❌ 不推荐

<div class="bg-white p-6 rounded-lg shadow-md border border-gray-200 hover:shadow-lg transition-shadow">
  <h3 class="text-lg font-semibold text-gray-900 mb-2">标题</h3>
  <p class="text-gray-600">内容...</p>
</div>

重复的类名组合，难以维护

✅ 推荐

.card {
  @apply bg-white p-6 rounded-lg shadow-md border border-gray-200 hover:shadow-lg transition-shadow;
}

<div class="card">
  <h3 class="text-lg font-semibold text-gray-900 mb-2">标题</h3>
  <p class="text-gray-600">内容...</p>
</div>

提取为可复用的组件类

工具类组织原则

推荐的类名顺序：

  1. 1. 布局类 (display, position, top, right, bottom, left)
  2. 2. 盒模型类 (width, height, margin, padding)
  3. 3. 视觉类 (background, border, shadow)
  4. 4. 排版类 (font, text, line-height)
  5. 5. 其他类 (transform, transition, cursor)

示例：

<div class="
  flex items-center justify-between
  w-full h-16
  bg-white border-b border-gray-200
  text-lg font-semibold text-gray-900
  transition-colors hover:bg-gray-50
">

2. 命名规范

组件命名规范

❌ 避免

  • • 使用通用名称 (box, item)
  • • 过于具体的名称 (blue-button-with-shadow)
  • • 缩写不清晰 (btn-prim, card-prod)
  • • 使用数字后缀 (button1, card2)

✅ 推荐

  • • 语义化名称 (primary-button, product-card)
  • • 功能描述 (submit-button, user-avatar)
  • • 清晰缩写 (btn-primary, card-product)
  • • 变体命名 (button-primary, button-secondary)

状态命名规范

状态类名应该清晰表达其用途：

交互状态
  • • hover:bg-blue-600
  • • focus:ring-2
  • • active:scale-95
  • • disabled:opacity-50
响应式状态
  • • sm:flex
  • • md:hidden
  • • lg:grid-cols-3
  • • xl:text-2xl
主题状态
  • • dark:bg-gray-800
  • • dark:text-white
  • • print:hidden
  • • landscape:flex-row

自定义类命名

使用 @apply 时的命名建议：

/* 基础组件 */
.btn {
  @apply px-4 py-2 rounded font-medium transition-colors;
}

/* 变体组件 */
.btn-primary {
  @apply btn bg-blue-500 hover:bg-blue-600 text-white;
}

.btn-secondary {
  @apply btn bg-gray-500 hover:bg-gray-600 text-white;
}

/* 状态组件 */
.btn-disabled {
  @apply btn opacity-50 cursor-not-allowed;
}

/* 尺寸组件 */
.btn-sm {
  @apply btn px-3 py-1 text-sm;
}

.btn-lg {
  @apply btn px-6 py-3 text-lg;
}

文档规范

组件文档模板：

/**
 * 主要按钮组件
 * 
 * @description 用于页面中的主要操作按钮
 * @example
 * <button class="btn-primary">提交</button>
 * 
 * @variants
 * - btn-secondary: 次要按钮
 * - btn-danger: 危险操作按钮
 * 
 * @sizes
 * - btn-sm: 小尺寸
 * - btn-lg: 大尺寸
 * 
 * @states
 * - disabled: 禁用状态
 * - loading: 加载状态
 */
.btn-primary {
  @apply bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded transition-colors;
}

4. 可维护性

组件抽象原则

❌ 过度抽象

.flexible-container {
  @apply flex items-center justify-center p-4 bg-white rounded shadow;
}

/* 过于通用，失去了语义化 */

✅ 适度抽象

.card {
  @apply bg-white rounded-lg shadow-md p-6;
}

.card-header {
  @apply border-b border-gray-200 pb-4 mb-4;
}

.card-content {
  @apply text-gray-600;
}

配置管理

环境配置分离：

// tailwind.config.base.js
module.exports = {
  content: ['./src/**/*.{html,js,jsx,ts,tsx}'],
  theme: {
    extend: {
      colors: {
        brand: {
          50: '#eff6ff',
          500: '#3b82f6',
          900: '#1e3a8a',
        }
      }
    }
  }
};

// tailwind.config.dev.js
const baseConfig = require('./tailwind.config.base.js');
module.exports = {
  ...baseConfig,
  plugins: [
    require('@tailwindcss/forms'),
    require('@tailwindcss/typography'),
  ]
};

// tailwind.config.prod.js
const baseConfig = require('./tailwind.config.base.js');
module.exports = {
  ...baseConfig,
  purge: {
    enabled: true,
    content: ['./src/**/*.{html,js,jsx,ts,tsx}'],
  }
};

5. 性能优化

构建优化

❌ 性能问题

  • • 未启用 PurgeCSS
  • • 包含未使用的样式
  • • 重复的类名定义
  • • 过大的 CSS 文件

✅ 优化方案

  • • 启用 PurgeCSS 清理
  • • 使用 CDN 缓存
  • • 压缩 CSS 文件
  • • 按需加载组件

运行时优化

CSS 优化

  • • 使用 transform 而非改变布局
  • • 避免频繁的 DOM 操作
  • • 使用 will-change 提示浏览器
  • • 优化动画性能

JavaScript 优化

  • • 使用事件委托
  • • 防抖和节流
  • • 虚拟滚动
  • • 懒加载组件

性能监控

关键性能指标：

LCP

最大内容绘制

目标: < 2.5s

FID

首次输入延迟

目标: < 100ms

CLS

累积布局偏移

目标: < 0.1

6. 常见陷阱和解决方案

CSS 特异性问题

❌ 问题

.my-component {
  @apply bg-red-500;
}

/* 外部样式覆盖 */
.my-component {
  background-color: blue !important;
}

使用 !important 破坏样式系统

✅ 解决方案

.my-component {
  @apply bg-red-500;
}

/* 使用 Tailwind 的响应式前缀 */
.my-component {
  @apply bg-red-500 md:bg-blue-500;
}

利用 Tailwind 的响应式系统

类名冲突

使用前缀避免冲突：

/* 使用项目前缀 */
.tw-btn-primary {
  @apply bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded;
}

.tw-card {
  @apply bg-white rounded-lg shadow-md p-6;
}

/* 或使用 BEM 命名 */
.btn--primary {
  @apply bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded;
}

.card__header {
  @apply border-b border-gray-200 pb-4 mb-4;
}

维护性问题

问题类型

  • • 魔法数字和硬编码值

  • • 缺乏文档和注释
  • • 不一致的命名规范

解决方案

  • • 使用设计令牌
  • • 提取可复用组件
  • • 编写详细文档
  • • 建立代码规范
